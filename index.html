HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The paragraph element is another kind of child elements inside of the body that represents a paragraph or a block of text.
  • The ul, ol and li elements represent unordered, ordered, and list elements.
  • The image (img) element contains information about images that are displayed on the webpage. The image element can contain various attributes such as the src attribute and the alt attribute.
  • The break (br) element creates a line of empty space or a line break between two blocks of content.
  • The anchor (a) element creates a link to te same webpage or other webpages.
  • the href attribute points to the URL for the link.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet and external CSS style sheet

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch name: creates a new branch and switches to it.
  • git add -a tells git to stage all files in order to record changes made to a file.
  • git commit -m "commit message": commits changes made in a branch. The commit message should be descriptive and must be in quotes.
  • git pull origin main: pulls the source code from GitHub and ensures that we have the latest commits and changes.
  • Pulling the source code before pushing up changes is a good practice to ensure there are no conflicts between GitHub and the local repo.
  • git push origin branch name: pushes changes to GitHub. Always push to your branch and now the main branch.
